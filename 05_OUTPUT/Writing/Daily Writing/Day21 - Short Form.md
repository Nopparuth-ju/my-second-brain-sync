
ผมเข้าใจเลยครับว่า การที่เราต้องซื้อ 'ที่เก็บข้อมูล' ทั้ง 2 ระบบพร้อมกัน มันอาจจะดูซ้ำซ้อนและสิ้นเปลืองงบ แต่ 2 ระบบนี้ มันมีความสำคัญและคุ้มค่าครับ อยากให้ลองนึกภาพว่ามันคล้ายกับการทำระบบน้ำประปา ดังนี้ครับ
data lake เปรียบเสมือน 'เขื่อนกักเก็บน้ำดิบ' ที่รองรับน้ำฝนหรือน้ำอื่นๆ ทุกรูปแบบ ข้อดีคือมันเก็บน้ำได้มหาศาลในราคาที่ถูก แต่ข้อเสียคือน้ำยังไม่สามารถนำมาดื่มได้ ในมุมของ data ก็คือไว้เก็บข้อมูลดิบๆที่ยังไม่พร้อมนำไปใช้สร้าง dashboard ครับ
ส่วน data warehouse ให้คิดว่าเป็นเหมือน 'แท็งก์เก็บน้ำสะอาดที่พร้อมดื่ม' ครับ โดยรับน้ำที่ผ่านเครื่องกรองมาแล้ว ในส่วนนี้จึงมีราคาแพงกว่า เราจึงต้องเก็บเฉพาะข้อมูลที่สะอาดและจัดระเบียบเรียบร้อยแล้ว เพื่อนำไปใช้สร้าง dashboard ที่ให้ข้อมูลแม่นยำและพร้อมใช้งานทันทีครับ
ดังนั้น ถ้าเรามีแค่ data lake ข้อมูลเราก็จะเละเทะจนนำไปตัดสินใจไม่ได้ แต่ถ้าเรามีแค่ data warehouse เราก็จะต้องเสียเงินมากเพื่อเก็บข้อมูล การซื้อทั้ง 2 ระบบให้ทำงานคู่กัน จึงเป็นการตัดสินใจที่เหมาะสมและคุ้มค่าครับ ไม่ใช่ความซ้ำซ้อน แต่เป็นวิธีการบริหารจัดการงบประมาณและข้อมูลให้คุ้มค่าที่สุดในระยะยาวครับผม
