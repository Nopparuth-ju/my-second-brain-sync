 
Deconstruct (ชำแหละด้วย First Principles): แยกให้ออกก่อนว่าสถานการณ์ตรงหน้า อะไรคือ "ข้อเท็จจริง" (Fact) และอะไรคือ "สมมติฐาน/ความรู้สึก" (Assumption)

Systems Mapping (มองความเชื่อมโยง): แทนที่จะแยกหัวข้อให้ขาดจากกัน ให้มองหา "ความสัมพันธ์" (อย่างที่คุณเห็นว่าการหักดิบวันนี้ จะกระทบความช่วยเหลือของ User ในอนาคต)

Root Cause Analysis (หาต้นตอ): เมื่อเห็นภาพรวมแล้ว จึงเจาะหาต้นตอด้วย 5 Whys (ถามทำไม 5 ครั้ง) เพื่อแก้ที่กระบวนการ ไม่ใช่แก้ที่คน

Second-Order Thinking (จำลองผลกระทบ): ก่อนเลือก Solution ให้ถามคำถามทองคำว่า "And then what? (ทำสิ่งนี้แล้ว จะเกิดอะไรตามมาอีก?)" เพื่อประเมินผลระยะสั้นและยาว
