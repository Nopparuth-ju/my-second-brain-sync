1.Timeless Knowledge (แก่นความรู้อมตะที่ต้องมี)
เป้าหมายของฝั่งนี้คือการเข้าใจ "ระบบ" และ "มนุษย์" หัวข้อที่คุณควรใช้เวลาศึกษาและย่อยลง 03_ZETTEL มีดังนี้ครับ:

1.1) System Thinking (การคิดเชิงระบบ): * Bottlenecks (ทฤษฎีคอขวด): หาจุดที่ทำให้ระบบช้าที่สุด (ประยุกต์ใช้ได้ตั้งแต่ Data Pipeline ไปจนถึงขั้นตอนจ่ายเงินของลูกค้า)
- Feedback Loops (วงจรผลป้อนกลับ): วงจรที่ทำให้เกิดการเติบโตแบบทวีคูณ (Reinforcing) หรือวงจรที่รักษาสมดุล (Balancing)

1.2) Behavioral Economics & Psychology (จิตวิทยาพฤติกรรมมนุษย์):
- Core Human Drives: แรงขับเคลื่อนพื้นฐาน (หาเงิน, หาความรู้, สุขภาพ, ความสัมพันธ์) ที่คุณได้จาก Personal MBA
- Friction & Incentive: การลดแรงเสียดทานในการใช้งาน และการสร้างแรงจูงใจให้คนยอมเปลี่ยนพฤติกรรม

1.3) Fundamental Business Mechanics (กลไกธุรกิจ):
- Value Creation vs. Value Delivery: การสร้างคุณค่า (หาปัญหาให้เจอ) และการส่งมอบ (เช่น การใช้ระบบ Cloud เข้ามาช่วยส่งมอบให้เร็วและถูกลง)
- Unit Economics: เข้าใจต้นทุนและกำไรต่อหน่วย (สำคัญมากเวลาทำ Web App หรือ Serverless เพราะต้องรู้ว่า Cost ต่อการรัน 1 ครั้งคุ้มกับที่ลูกค้าจ่ายไหม)

2.Bleeding Edge Knowledge (ความรู้หน้าผา/เทคโนโลยีล่าสุด)
เป้าหมายของฝั่งนี้คือ "เครื่องมือและทักษะ" ที่นำมาประกอบร่าง (Combinatorial) เพื่อแก้ปัญหาทางธุรกิจให้เร็วที่สุด โฟกัสไปที่สิ่งที่คุณสามารถ "ลงมือทำและทดสอบ" ได้จริง:

2.1) Modern Data Stack & Architecture:
- Serverless & Edge Computing: การออกแบบสถาปัตยกรรมที่ไม่ต้องจัดการ Server เอง เน้นความเร็ว (เช่น การใช้ Worker, การจัดการ Object Storage)
- Cloud Data Platforms: การจัดการ Data Warehouse บน Cloud สมัยใหม่ (เช่น โครงสร้างและการคิวรีข้อมูลในระบบระดับองค์กร)

2.2) AI Integration & Agentic Workflows:
- LLMs as Reasoning Engines: เลิกมอง AI เป็นแค่แชทบอท แต่ใช้มันเป็น "เอ็นจิ้นคิดวิเคราะห์"
- AI-Powered Applications: การเชื่อม API เพื่อสร้างแอปพลิเคชันที่มี AI ฝังอยู่เบื้องหลัง (เช่น การต่อฐานข้อมูลบน Cloud เข้ากับ AI เพื่อดึงข้อมูลมาประมวลผลทันที)

2.3) Data Storytelling & Visualization:
- การแปลงข้อมูลดิบจากสถาปัตยกรรมหลังบ้าน ให้กลายเป็น Dashboard หรือ Report ที่คนอ่านแล้วเกิด Action ทันที (ลด Noise เพิ่ม Signal)
